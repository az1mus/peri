# MCP 初始化失败后状态栏红色错误提示无法消失

**状态**：Verified
**优先级**：中
**创建日期**：2026-06-16

## 问题描述

MCP 服务器连接失败后，底部状态栏显示红色「MCP 失败: N 个服务器连接失败」文字。该错误提示一旦出现就无法自动消失，即使后续网络恢复、或用户在 MCP 面板中对单个服务器执行重连操作，状态栏红字仍然持续显示，直到重启应用。

## 症状详情

| 维度 | 现象 |
|------|------|
| **触发条件** | MCP 初始化时部分或全部服务器连接失败 |
| **实际行为** | 状态栏持续显示红色「MCP 失败: N 个服务器连接失败」，应用重启前永不消失 |
| **期望行为** | 用户重连成功后错误提示应消失，或至少提供手动关闭/超时自动消失机制 |
| **复现频率** | 必现（只要存在一个无法连接的 MCP 服务器） |

## 复现条件

- **复现频率**：必现
- **触发步骤**：
  1. 配置一个无法连接的 MCP 服务器（如指向不存在的地址）
  2. 启动 peri-tui
  3. 观察状态栏：初始化完成后显示红色「MCP 失败: 1 个服务器连接失败: xxx: ...」
  4. 该红字在应用运行期间不会消失（重启应用、重新初始化 MCP 才能清除）
- **环境**：不限

## 涉及文件

- `peri-middlewares/src/mcp/client.rs` —— `McpInitStatus` 枚举定义，`Failed` 为终端状态无自动恢复
- `peri-tui/src/ui/main_ui/status_bar.rs` —— 状态栏渲染 `McpInitStatus::Failed` 红色文字
